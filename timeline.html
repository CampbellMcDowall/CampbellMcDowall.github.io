Timeline

Week One

Tasks complete/in progress

Set up the Multitech Conduit at home, this way it would be able to be configured off of the Otago Polytechnics network. Previously the network had proved troublesome with various equipment and the first test of the gateway at Polytech. As I set it up I also started documentation/rough notes.

Issues/Challenges

At first there were a few issues with the devices set up, these mainly were due to lack of up to date concrete documentation. I needed to change my IP subnet to access the device for the first time.

Week Two

Tasks complete/in progress

After the first week of trying to set up the gateway I continued to try get it to work correctly, this took a number of factory resets and a bit of trial and error to get the device to behave correctly. I then worked on getting Node-RED working and the flow for connecting the Multitech mDot Box to the gateway.

Issues/Challenges

Found out that the mDot Box was setup for US frequencies by default and ended up spending a while writing emails to the technical support about how to change the mDot Box firmware to the correct AU915 frequency

Week Three

Tasks complete/in progress

At the end of week 2 I received some firmware and instructions from the technical support team, I spent the first part of the week learning how to use the USB Micro Development board and the AT commands to communicate with the device. Continued with documentation. Nearing the end of the week I received updated firmware and instructions to change the firmware. Over the weekend and the Monday of week 4 I finally got the firmware changed on the mDot Box.

Issues/Challenges

Unfortunately the first lot of firmware and instructions were not suitable for changing the mDot Box firmware. The process was a bit confusing due to their being so many firmware versions available and they needed to be installed in a specific order and the mDot needed to use a specific version before I could change the frequency. Changing the firmware and frequencies seemed to have worked, although there is sometimes an error in the serial command.
